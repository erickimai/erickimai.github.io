Enviando meu primeiro patch para o kernel Linux

# contexto

A mudança em si foi simples — substituir chamadas mutex_lock() e mutex_unlock() pelo macro guard(mutex)() no driver m62332, usando os Scope-based Cleanup Helpers do kernel.

# a batalha com o email

A parte técnica do patch foi tranquila. O que me tomou mais tempo foi configurar o git send-email para funcionar com o email da USP. O problema central é que o Perl usado pelo git send-email não possui suporte nativo ao mecanismo OAUTHBEARER — o módulo simplesmente não existe no CPAN.

A solução foi usar o msmtp como cliente SMTP no lugar do envio direto pelo Perl. O msmtp tem suporte nativo a OAuth2, então bastou configurá-lo para buscar o token via git-credential-gmail e apontar o git send-email para ele com sendmailcmd = msmtp. Depois disso, funcionou perfeitamente.

# resultado

O patch passou pelo CI interno do curso sem erros e foi enviado para os mantenedores do subsistema IIO e para a mailing list linux-iio@vger.kernel.org.
